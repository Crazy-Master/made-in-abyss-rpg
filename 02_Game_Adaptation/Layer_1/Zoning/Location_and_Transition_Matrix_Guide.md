# Location and Transition Matrix Guide (Руководство по матрицам мест и переходов)

## Кратко

Матрицы мест и переходов нужны не как финальные документы, а как проектная сетка.

Они помогают быстро увидеть:

```
какие места есть;
зачем они нужны;
какие переходы их связывают;
где main_route;
где side_route;
где hidden;
где флора, находки, фауна, риск и возврат.
```

Матрицы создаются до полноценных карточек мест и переходов.

Главная задача матриц:

```
Быстро спроектировать структуру участка, не расписывая каждую сцену полностью.
```

## Когда использовать матрицы

Матрицы используются после зонирования и до карточек.

Рабочий порядок:

```
1. Зонирование слоя.
2. Выбор зоны или подзоны для разработки.
3. Матрица мест.
4. Матрица переходов.
5. Проверка связей.
6. Полноценные карточки мест и переходов.
```

Для первого учебного спуска матрицы должны описать только рабочий участок, а не весь первый слой.

## Матрица мест

Матрица мест отвечает на вопрос:

```
Какие сцены-узлы есть в маршруте и зачем они нужны?
```

Место — это открытая или условно открытая сцена исследования и выбора. Внутри места перемещение не является главным испытанием.

Матрица места должна быть короткой. Она не заменяет карточку места.

## Поля матрицы мест

| Поле               | Что указывать                                                                              |
| ------------------ | ------------------------------------------------------------------------------------------ |
| `location_id`      | ID будущей карточки места, например `L001`.                                                |
| `name`             | Английское техническое название.                                                           |
| `ru_name`          | Русское название для чтения.                                                               |
| `zone_id`          | Зона или подзона, к которой относится место.                                               |
| `height_m`         | Высота от поверхности в метрах.                                                            |
| `survey_level`     | Насколько место изучено Delvers.                                                           |
| `availability`     | Роль места: основное, боковое, скрытое, опасное и т.д.                                     |
| `spatial_type`     | Тип пространства: уступ, поляна, трещина, стоянка и т.д.                                   |
| `function`         | Зачем это место нужно в игре.                                                              |
| `visible_features` | Что заметно сразу при входе в место.                                                       |
| `hidden_features`  | Что скрыто или требует внимания.                                                           |
| `flora`            | Важная флора, если она влияет на сцену.                                                    |
| `fauna`            | Фауна или следы. Пока можно писать обобщённо.                                              |
| `relics_finds`     | Реликвии, предметы, образцы, записи, следы.                                                |
| `risks`            | Основные риски места.                                                                      |
| `transition_notes` | Заметки о нужных переходах; точные связи фиксируются в матрице переходов через `connects`. |
| `card_notes`       | Заметки для будущей карточки места.                                                        |

## Значения `survey_level`

`survey_level` показывает, насколько участок изучен Delvers.

```
mapped — хорошо нанесено на карты и часто используется;
patrolled — регулярно посещается или проверяется;
known — известно, но не контролируется;
rough — описано плохо;
wild — почти не изучено;
forbidden — опасно, закрыто или формально запрещено.
```

Для первого учебного спуска чаще всего используются:

```
mapped;
patrolled;
known;
rough.
```

`wild` и `forbidden` лучше оставить для более поздних или опасных участков.

## Значения `availability`

`availability` показывает роль места в структуре.

```
main_route — основное место маршрута;
side_route — боковая зона или боковой путь;
hidden — скрытое место;
optional — необязательное место;
dangerous — опасное место;
special — сюжетно или событийно важное место.
```

Если место одновременно боковое и опасное, лучше выбрать главное значение, а второе указать в `risks` или `card_notes`.

## Значения `spatial_type`

Можно использовать те же значения, что в карточке места:

```
ledge — уступ;
shelf — широкая полка;
clearing — открытая площадка / поляна;
hollow — ниша / углубление;
crack — трещина;
cave — пещерная зона;
slope_base — основание склона;
slope_top — верх склона;
water_edge — край воды / ручья / водопада;
root_platform — корневая площадка;
ruin_fragment — фрагмент старой структуры;
camp_site — стоянка;
flower_field — цветочное поле;
stone_field — каменное поле / россыпь.
```

Если точного варианта нет, можно временно написать своё значение и позже нормализовать.

# Матрица переходов

Матрица переходов отвечает на вопрос:

```
Какие сцены-пути соединяют места и что в них важно?
```

Переход — это направленная сцена перемещения между местами или между другими переходами.

Главная формула:

```
Переход — это туннель из точек.
```

Матрица перехода не должна описывать весь переход полностью. Она только фиксирует структуру будущей карточки.

## Поля матрицы переходов

|Поле|Что указывать|
|---|---|
|`transition_id`|ID будущей карточки перехода, например `T001`.|
|`name`|Английское техническое название.|
|`ru_name`|Русское название для чтения.|
|`connects`|Какие места или переходы соединяет.|
|`zone_id`|Основная зона или подзона.|
|`height_a_m`|Высота первой связанной точки.|
|`height_b_m`|Высота второй связанной точки.|
|`height_delta_m`|Перепад высоты.|
|`survey_level`|Насколько переход изучен Delvers.|
|`availability`|Роль перехода.|
|`difficulty`|Общая сложность для мастера.|
|`concept_hint`|Идея перехода: спуск, карниз, трещина и т.д.|
|`passage_points`|Краткий список точек прохождения.|
|`hidden_branches`|Скрытые ответвления, если есть.|
|`state_flags`|Что может измениться после действий игроков.|
|`risks`|Основные риски перехода.|
|`card_notes`|Заметки для будущей карточки перехода.|

## Значения `difficulty`

`difficulty` — это не сложность проверки, а общая оценка для мастера.

```
low — простой переход, риск в основном из-за глупых действий;
moderate — заметный риск, нужно думать о снаряжении или условиях;
high — опасный переход, ошибка может иметь серьёзные последствия;
extreme — переход для очень опасных участков, не для первого учебного спуска.
```

Для первого учебного спуска лучше использовать:

```
low;
moderate.
```

`high` можно дать одному боковому или скрытому переходу, если он явно не предназначен для безопасного учебного пути.

## `concept_hint`

`concept_hint` — это не строгая классификация, а подсказка для разработки.

Возможные идеи:

```
спуск;
подъём;
карниз;
трещина;
тоннель;
склон;
осыпь;
корневой путь;
верёвочный спуск;
заросший проход;
обход по стене;
переправа;
путь по старой конструкции;
ход через пещеру;
узкий лаз;
путь вдоль воды.
```

Переход может сочетать несколько идей:

```
влажная трещина + боковой обход;
верёвочный спуск + старый каменный выступ;
карниз + следы мелкой фауны.
```

## `passage_points`

`passage_points` — это краткий список будущих точек прохождения.

Писать коротко:

```
верхний край; влажный выступ; нижняя стенка
```

или:

```
узкий вход; моховой изгиб; низкий проём; выход к полке
```

Каждая точка потом станет фрагментом полноценной карточки перехода.

## `state_flags`

`state_flags` фиксирует, что может измениться.

Примеры:

```
верёвка оборвана;
край осыпался;
путь стал скользким;
обратный путь стал опаснее;
фауна привлечена;
метка стёрта;
боковой проход открыт;
образцы у воды испорчены.
```

Не нужно писать много флагов. Обычно достаточно 2–4.

# Проверка матриц

После заполнения матриц нужно проверить структуру.

## Проверка мест

В матрице мест должно быть понятно:

```
зачем существует каждое место;
какое место является стартовым;
где можно остановиться;
где можно собрать образцы;
где есть риск;
где есть скрытые или боковые элементы;
как место связано с переходами.
```

Если место не имеет функции, его нужно убрать, объединить с соседним или превратить в точку прохождения перехода.

## Проверка переходов

В матрице переходов должно быть понятно:

```
что соединяет переход;
почему это не просто прямая линия;
какие точки прохождения в нём есть;
что может измениться после действий игроков;
есть ли риск при возвращении;
есть ли скрытые ответвления.
```

Если переход слишком пустой, его можно сделать простым связующим переходом или объединить с соседним.

Если переход слишком насыщенный, его лучше разделить на несколько переходов и промежуточное место.

## Проверка графа

Минимальная проверка:

```
из стартового места можно попасть в основную зону;
есть путь вниз;
есть путь обратно;
есть хотя бы одно боковое ответвление;
есть хотя бы один скрытый элемент;
нет бессмысленных тупиков;
опасные переходы не обязательны для первого учебного пути;
места с важными задачами достижимы разными логичными способами или достаточно явно читаются.
```

## Проверка первого учебного спуска

Для первого учебного спуска желательно иметь:

```
8–12 мест;
10–16 переходов;
1 основной учебный путь;
2–4 боковых ответвления;
1–2 скрытых перехода;
2–3 точки сбора образцов;
1 точку отправки шара;
1 точку отметки успешного спуска;
1 важную необычную находку;
1 мягкое проявление Проклятия на обратном пути.
```

Это ориентир, а не жёсткое правило.

# Что не нужно делать в матрицах

В матрицах не нужно:

```
писать полноценные описания мест;
писать полные точки внимания;
расписывать все проверки;
расписывать все варианты действий игроков;
делать художественный текст;
создавать финальную карту.
```

Матрица должна быть компактной.

Если строка стала слишком длинной, значит это уже материал для будущей карточки, а не для матрицы.

## Главное правило

```
Матрица должна помогать быстро проектировать структуру.

Карточка должна помогать вести сцену.

Не смешивать эти две задачи.
```
