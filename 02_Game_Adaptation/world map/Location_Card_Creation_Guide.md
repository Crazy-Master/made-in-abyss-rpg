# Location Card Creation Guide (Руководство по карточке места)

## Кратко

Этот документ описывает формат карточки места для `MadeInAbyss_RPG`.

Карточка места нужна мастеру как рабочая сцена: описание для игры, скрытая информация, точки внимания, состояние места и переходы.

Место — это открытая или условно открытая область, где персонажи могут относительно свободно перемещаться по горизонтали.

Главный признак места:

Внутри места перемещение не является главным испытанием.

Если зона становится слишком большой, сложной или насыщенной событиями, её лучше разделить на несколько мест и соединить переходами.

## Основной принцип

Карточка места не должна выглядеть как меню действий.

Игроки не должны получать список:

```
Здесь есть растение, переход, реликвия и ловушка.
```

Вместо этого важные элементы вплетаются в описание места:

```
Среди обычной травы у края уступа выделяются серо-зелёные стебли с загнутыми кончиками ([[Hookgrass|Кагикуса]]). Левее площадка обрывается вниз, и из трещины в камне свисит старая потемневшая верёвка ([[T001_Old_Rope_Drop|старый верёвочный спуск]]).
```

Мастер видит ссылки и понимает структуру, а игроки воспринимают это как живое описание сцены.

## Папка

Карточки мест хранятся по слоям.

Пример:

```
02_Game_Adaptation/Layer_1/Locations/
02_Game_Adaptation/Layer_2/Locations/
02_Game_Adaptation/Layer_3/Locations/
```

## Наименование файла

Место получает ID и короткое художественное название.

Формат:

```
L001_Old_Training_Ledge.md
L002_Wet_Crack.md
L003_Amberberry_Shelf.md
```

`L` означает `location`.

Число даёт уникальность.

Название должно быть художественным, но понятным.

## YAML

Пример YAML:

```
---
type: location
status: draft
data_layer: game_adaptation
layer: Layer_1
location_id: L001
height_m: 120

spatial_type: ledge

  - route_node

  - Layer_1
  - Layer_1_5_Environment_Reference

transitions:
  - T001_Old_Rope_Drop
  - T002_Mossy_Crack_Passage

  - Dewcup_Moss
fauna:
  - small_fauna
relics:
  - Star_Compass
created: 2026-06-26
updated: 2026-06-26
---
```

## YAML-поля

`type` — всегда `location`.

`status` — рабочий статус карточки. Обычно `draft`, позже можно менять.

`data_layer` — для игровых мест используем `game_adaptation`.

`layer` — слой Бездны.

`location_id` — ID места.

`height_m` — высота в метрах от поверхности.

`availability` — роль места в маршруте.

Возможные значения:

```
main_route — основное место маршрута
side_route — боковая зона
hidden — скрытое место
optional — необязательная зона
dangerous — опасная зона
special — сюжетно или событийно важное место
```

`spatial_type` — тип пространства.

Возможные значения:

```
ledge — уступ
shelf — широкая полка
clearing — открытая площадка / поляна
hollow — ниша / углубление
crack — трещина
cave — пещерная зона
slope_base — основание склона
slope_top — верх склона
water_edge — край воды / ручья / водопада
root_platform — корневая площадка
ruin_fragment — фрагмент старой структуры
camp_site — стоянка
flower_field — цветочное поле
stone_field — каменное поле / россыпь
```

`transitions` — переходы, связанные с местом.

`flora` — важная флора места. Обычную траву, кусты и фоновые растения сюда не заносить.

`fauna` — возможная фауна. Пока конкретная фауна не создана, можно писать `small_fauna`, `insects`, `small_birds`, `fauna_traces`.

`relics` — реликвии или важные находки, если они есть.

## Заголовок

Формат:

```
# L001 Old Training Ledge (Старый учебный уступ)
```

Сразу под заголовком указываются высота, положение и возможная фауна.

Пример:

```
Высота: 120 м от поверхности.

Положение: широкий каменный уступ у влажной стены Бездны.

Возможная фауна: следы мелкой фауны, насекомые, мелкие птицы.
```

Положение пишется на русском, коротко и понятно. Оно дублирует `spatial_type`, но в нормальном человеческом виде.

## Кратко

Короткий блок для быстрого понимания места.

Пример:

```
## Кратко

Старый учебный уступ на первом слое. Подходит для короткой остановки, первого осмотра маршрута и выбора дальнейшего спуска.
```

Блок должен отвечать на вопросы:

```
Что это за место?
Зачем оно нужно в игре?
Какую функцию выполняет?
```

## Описание места

Главный описательный блок.

Пишется нейтрально-описательным стилем, без прямого обращения к игрокам.

Хорошо:

```
С уступа открывается широкий вид на верхние стены Бездны.
```

Хуже:

```
Вы видите широкий вид на верхние стены Бездны.
```

Описание должно быть красочным, но не перегруженным.

В описание можно вплетать:

```
флору;
следы фауны;
переходы;
реликвии;
предметы;
старые лагеря;
ориентиры;
опасные детали;
скрытые намёки.
```

Важно: не раскрывать всё прямо. Игроки должны сами обращать внимание на детали.

Пример:

```
## Описание места

С уступа открывается широкий вид на верхние стены Бездны: камень уходит вниз неровными серыми ступенями, а далеко внизу слышен глухой шум воды. Поверхность площадки достаточно ровная, чтобы здесь можно было остановиться и разложить снаряжение, но у края камень темнее и влажнее, чем в центре.

У правой стороны уступа обычная трава становится жёстче: среди зелени выделяются серо-зелёные стебли с загнутыми кончиками ([[Hookgrass|Кагикуса]]). Ближе к влажной стене в трещинах поблёскивают крошечные чашечки мха ([[Dewcup_Moss|Цуюмох]]), в которых держатся прозрачные капли воды. Слева старый край площадки обрывается вниз; там, где камень уходит за уступ, из трещины свисит потемневшая верёвка ([[#У старой верёвки|старая верёвка]]).
```

## Альтернативные описания в основном блоке

Если альтернативное описание уже нужно, его можно добавить прямо под базовым описанием через callout.

Используем стандартные callouts Obsidian:

```
> [!info] Ночью
> Ночная версия описания.

> [!success] Спокойное развитие
> Безопасная или дружелюбная версия сцены.

> [!danger] Опасное развитие
> Версия с угрозой, засадой, ловушкой или осложнением.
```

Базовое описание остаётся обычным текстом. Callouts нужны только для альтернатив.

Пример:

```
> [!info] Ночью
> В темноте влажная стена становится заметнее: между трещинами проступает слабое зеленоватое свечение грибного налёта.
```

## Для мастера

Короткая расшифровка сцены.

Здесь не нужно повторять описание. Нужно кратко объяснить, что реально важно.

Пример:

```
## Для мастера

Явное место остановки и выбора пути. Здесь есть старый лагерь, видимый спуск по верёвке, влажная стена с мхом и участок [[Hookgrass|Кагикуса]] у края. Шум или раздавленные ягоды могут привлечь мелкую фауну.
```

Этот блок должен быть коротким, логичным и практичным.

## Состояние места

Состояние места указывается только если его реально нужно отслеживать.

Состояние места — это не атмосфера, а то, что мастер должен помнить после действий игроков.

Пример:

```
Состояние места:

- [ ] Лагерь обыскан.
- [ ] Верёвка у края потревожена.
- [ ] Флора у края повреждена.
- [ ] Место стало заметным для фауны.
```

Если место простое и не меняется, блок состояния можно не добавлять.

## Точки внимания

Точки внимания — главный инструмент для скрытых деталей, ловушек, находок и малых сцен внутри места.

Каждая точка внимания должна иметь понятное название:

```
## Точки внимания

### У старой верёвки

...

### У Кагикуса

...

### У влажной стены

...
```

Название должно сразу объяснять, куда смотреть мастеру.

Если в основном описании есть ссылка на точку внимания, лучше ссылаться на заголовок:

```
[[#У старой верёвки|старая верёвка]]
```

Это удобнее, чем ссылки на отдельные слова или блоки.

## Формат точки внимания

Каждая точка внимания строится так:

```
### Название точки

Описание того, что замечают персонажи при подходе, осмотре или взаимодействии.

Если персонажи делают конкретное действие, описывается реакция среды или последствия.

**Для мастера:** короткая расшифровка: что здесь скрыто, какие условия, какие последствия.
```

Пример:

```
### У старой верёвки

Верёвка висит неподвижно, но выглядит слишком старой для надёжного спуска. Верхний узел забит в каменную трещину, рядом с ним видны сухие волокна и мелкая каменная крошка.

Если потянуть за верёвку сильнее, сверху осыпается пыль и несколько маленьких осколков.

Если персонажи внимательно проверяют верёвку, становится понятно, что она выдержит лёгкое натяжение, но полноценный спуск по ней опасен.

**Для мастера:** верёвка связана с [[T001_Old_Rope_Drop]]. При грубом использовании можно отметить состояние перехода “верхний зацеп повреждён” или “обратный путь стал опаснее”.
```

## Альтернативы внутри точек внимания

Если альтернативное состояние влияет только на конкретную точку, оно пишется внутри этой точки.

Пример:

```
### У Кагикуса

Жёсткая трава у края уступа цепляется за ткань и ремни. Между её стеблями заметны следы, будто по траве недавно протащили небольшой предмет.

> [!info] Ночью
> В темноте сами стебли почти неразличимы, но между ними иногда поблёскивает слабый металлический отблеск.

**Для мастера:** находка появляется только при внимании к траве, сборе образцов или подробном осмотре края.
```

## Ловушки

Ловушки оформляются как точки внимания, а не как отдельный сухой список.

Ловушка должна иметь видимые признаки.

Хорошо:

```
### У старой верёвки

Верёвка выглядит старой, волокна у верхнего узла распушены, а камень вокруг крепления крошится.

Если потянуть за верёвку сильнее, сверху осыпается мелкая каменная крошка.
```

Плохо:

```
Ловушка: верёвка может оборваться.
```

Принцип:

```
Сначала видимые признаки.
Потом действие игроков.
Потом последствия.
```

## Альтернативные состояния

Этот блок находится ближе к концу карточки, перед переходами.

Он нужен как краткая памятка будущих альтернатив, если они ещё не расписаны полностью.

Пример:

```
## Альтернативные состояния

Ночью: заметнее влажная стена и светящиеся налёты.  
Во время дождя: край уступа становится скользким.  
При событии: здесь могут быть другие Delvers.
```

Если альтернативы уже подробно описаны в `Описание места` или `Точки внимания`, здесь можно оставить только краткое напоминание.

## Переходы

Блок переходов всегда находится в самом низу карточки.

Он обязателен даже если переход один.

Пример:

```
## Переходы

[[T001_Old_Rope_Drop]] — видимый спуск с края уступа. Быстрый путь вниз, но верёвка ненадёжна.

[[T002_Mossy_Crack_Passage]] — боковой путь через влажную трещину у стены. Становится заметен при осмотре [[Dewcup_Moss|Цуюмха]] или поиске воды.
```

Переходы должны быть краткими. Подробности находятся в карточках самих переходов.

## Таблицы

Таблицы не являются обязательной частью карточки места.

Их можно использовать только если место сложное и текст становится неудобным.

Подходящие случаи:

```
много переходов;
много находок;
сложное состояние места;
несколько вариантов фауны или событий.
```

Если место простое, лучше писать обычным текстом.

# L001 Old Training Ledge (Старый учебный уступ)

Высота: 120 м от поверхности.

Положение: широкий каменный уступ у влажной стены Бездны.

Возможная фауна: следы мелкой фауны, насекомые, мелкие птицы.

## Кратко

Старый учебный уступ на первом слое. Подходит для короткой остановки, первого осмотра маршрута и выбора дальнейшего спуска.

## Описание места

С уступа открывается широкий вид на верхние стены Бездны: камень уходит вниз неровными серыми ступенями, а далеко внизу слышен глухой шум воды. Поверхность площадки достаточно ровная, чтобы здесь можно было остановиться и разложить снаряжение, но у края камень темнее и влажнее, чем в центре.

У правой стороны уступа обычная трава становится жёстче: среди зелени выделяются серо-зелёные стебли с загнутыми кончиками ([[Hookgrass|Кагикуса]]). Ближе к влажной стене в трещинах поблёскивают крошечные чашечки мха ([[Dewcup_Moss|Цуюмох]]), в которых держатся прозрачные капли воды. Слева старый край площадки обрывается вниз; там, где камень уходит за уступ, из трещины свисит потемневшая верёвка ([[#У старой верёвки|старая верёвка]]).

> [!info] Ночью
> В темноте влажная стена становится заметнее: между трещинами проступает слабое зеленоватое свечение грибного налёта.

## Для мастера

Явное место остановки и выбора пути. Здесь есть старый лагерь, видимый спуск по верёвке, влажная стена с мхом и участок [[Hookgrass|Кагикуса]] у края. Шум или раздавленные ягоды могут привлечь мелкую фауну.

Состояние места:

- [ ] Лагерь обыскан.
- [ ] Верёвка у края потревожена.
- [ ] Флора у края повреждена.
- [ ] Место стало заметным для фауны.

## Точки внимания

### У старой верёвки

Верёвка висит неподвижно, но выглядит слишком старой для надёжного спуска. Верхний узел забит в каменную трещину, рядом с ним видны сухие волокна и мелкая каменная крошка.

Если потянуть за верёвку сильнее, сверху осыпается пыль и несколько маленьких осколков.

Если персонажи внимательно проверяют верёвку, становится понятно, что она выдержит лёгкое натяжение, но полноценный спуск по ней опасен.

**Для мастера:** верёвка связана с [[T001_Old_Rope_Drop]]. При грубом использовании можно отметить состояние перехода “верхний зацеп повреждён” или “обратный путь стал опаснее”.

### У Кагикуса

Жёсткая трава у края уступа цепляется за ткань и ремни. Между её стеблями заметны следы, будто по траве недавно протащили небольшой предмет. Если персонажи подходят ближе или собирают образцы, среди спутанных стеблей можно заметить слабый металлический блеск.

При аккуратном осмотре в траве находится маленький предмет, почти скрытый землёй и стеблями ([[Star_Compass|Звёздный компас]]). Грубый сбор травы может сдвинуть находку ближе к краю.

> [!info] Ночью
> В темноте сами стебли почти неразличимы, но между ними иногда поблёскивает слабый металлический отблеск.

**Для мастера:** находка появляется только при внимании к траве, сборе образцов или подробном осмотре края. Если персонажи действуют грубо, компас не исчезает, но его извлечение становится сложнее.

### У влажной стены

В трещинах стены растёт [[Dewcup_Moss|Цуюмох]]. Камень вокруг прохладный и тёмный, а в чашечках мха собирается вода. Ниже по стене заметна узкая влажная линия, уходящая в боковую расщелину.

Если персонажи изучают мох или ищут источник влаги, они могут заметить начало узкого обходного пути ([[T002_Mossy_Crack_Passage|влажная трещина]]).

**Для мастера:** это мягкая подсказка на боковой переход. Вода из мха требует фильтрации или кипячения, если персонажи хотят пить её без риска.
