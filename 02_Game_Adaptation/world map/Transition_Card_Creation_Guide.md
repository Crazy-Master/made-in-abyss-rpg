# Transition Card Creation Guide (Руководство по карточке перехода)

## Кратко

Этот документ описывает формат карточки перехода для `MadeInAbyss_RPG`.

Переход — это направленная сцена перемещения между местами или между другими переходами.

Главная формула:

Переход — это туннель из точек.

Это не значит, что переход обязательно является тоннелем. Это значит, что персонажи двигаются через ограниченное пространство и последовательно проходят несколько точек.

## Чем переход отличается от места

Место — открытая сцена исследования и выбора.

Переход — направленная сцена движения.

В месте персонажи могут свободно перемещаться по горизонтали. В переходе форма пути ограничивает движение: склон, карниз, трещина, верёвочный спуск, корневой проход, тоннель, заросший лаз, осыпь или узкий обход.

## Основной принцип

Карточка перехода не должна перечислять все возможные способы прохождения.

Игроки могут придумать больше вариантов, чем указано в документе. Поэтому карточка описывает не “разрешённые действия”, а:

```
среду;
точки прохождения;
условия;
последствия;
состояние перехода;
скрытые ответвления;
направления движения.
```

## Папка

Карточки переходов хранятся по слоям.

Пример:

```
02_Game_Adaptation/Layer_1/Transitions/
02_Game_Adaptation/Layer_2/Transitions/
02_Game_Adaptation/Layer_3/Transitions/
```

## Наименование файла

Переход получает ID и короткое понятное название.

Формат:

```
T001_Old_Rope_Drop.md
T002_Mossy_Crack_Passage.md
T003_Outer_Stone_Traverse.md
```

`T` означает `transition`.

Число даёт уникальность.

Название описывает характер пути, но не обязано включать оба связанных места.

Не использовать слишком длинные названия вида:

```
Old_Training_Ledge_to_Wet_Crack_Slippery_Root_Descent.md
```

Связанные места указываются в YAML и теле карточки.

## YAML

Пример YAML:

```
---
type: transition
status: draft
data_layer: game_adaptation
layer: Layer_1
transition_id: T001
height_a_m: 120
height_b_m: 145
height_delta_m: 25
availability: main_route
difficulty: low
source:
  - project_notes
tags:
  - transition
  - layer_1
  - route_edge
connects:
  - L001_Old_Training_Ledge
  - L002_Lower_Wet_Ledge
related:
  - Layer_1
  - Location_and_Transition_Principles
  - Location_Card_Creation_Guide
created: 2026-06-26
updated: 2026-06-26
---
```

## YAML-поля

`type` — всегда `transition`.

`status` — рабочий статус карточки. Обычно `draft`, позже можно менять.

`data_layer` — для игровых переходов используем `game_adaptation`.

`layer` — слой Бездны.

`transition_id` — ID перехода.

`height_a_m` — высота первой связанной точки.

`height_b_m` — высота второй связанной точки.

`height_delta_m` — перепад высоты между связанными точками. Если переход почти горизонтальный, можно указать `0` или малое значение.

`availability` — роль перехода в маршруте.

Возможные значения:

```
main_route — основной переход маршрута
side_route — боковой переход
hidden — скрытый переход
optional — необязательный переход
dangerous — опасный переход
special — сюжетно или событийно важный переход
```

`difficulty` — общая ориентировочная сложность для мастера, не конкретная сложность проверки.

Возможные значения:

```
low
moderate
high
extreme
```

`connects` — связанные места или переходы.

Пример:

```
connects:
  - L001_Old_Training_Ledge
  - L002_Lower_Wet_Ledge
```

Не используем `from` и `to`, потому что переход может проходиться в обе стороны.

## Типы переходов

Строгую классификацию переходов в YAML не используем.

Переходы должны оставаться индивидуальными.

Но при создании матриц и идей можно опираться на такие типы:

```
спуск;
подъём;
карниз;
трещина;
тоннель;
склон;
осыпь;
корневой путь;
верёвочный спуск;
заросший проход;
обход по стене;
переправа;
путь по старой конструкции;
ход через пещеру;
узкий лаз;
путь вдоль воды.
```

Это не обязательные категории, а справочник идей.

## Заголовок

Формат:

```
# T001 Old Rope Drop (Старый верёвочный спуск)
```

Сразу под заголовком указываются высота, перепад и связи.

Пример:

```
Высота: между 120 м и 145 м от поверхности.

Перепад: около 25 м.

Соединяет [[L001_Old_Training_Ledge]] и [[L002_Lower_Wet_Ledge]].
```

## Кратко

Короткий блок для быстрого понимания перехода.

Пример:

```
## Кратко

Переход между двумя уступами. Сверху вниз он ощущается как быстрый спуск, снизу вверх — как медленный подъём по влажной стене.
```

Блок должен отвечать на вопросы:

```
Что это за переход?
Какие места он соединяет?
Почему он важен?
```

## Для мастера

Короткая расшифровка перехода.

Пример:

```
## Для мастера

Старая верёвка ненадёжна. Большой предмет мешает на влажном выступе. Если повредить верхний край или оборвать верёвку, обратный путь станет опаснее.
```

Этот блок должен быть коротким, логичным и практичным.

## Состояние перехода

Состояние перехода почти всегда полезно, потому что переходы могут меняться после действий игроков.

Пример:

```
## Состояние перехода

- [ ] Верёвка оборвана.
- [ ] Верхний край осыпался.
- [ ] Влажный выступ стал скользким.
- [ ] Переход привлёк мелкую фауну.
```

Чекбоксов не должно быть слишком много. Обычно достаточно 2–4.

Состояние перехода — это не атмосфера, а то, что мастер должен помнить после действий игроков.

## Опорная логика перехода

Этот блок заменяет “варианты прохождения”.

Здесь не перечисляются все возможные действия игроков. Здесь указываются общие правила среды и последствия обобщённых действий.

Пример:

```
## Опорная логика перехода

Старая верёвка показывает прежний маршрут, но не выдерживает резкой нагрузки.

Собственная верёвка помогает, если персонажи нашли надёжное крепление.

Большой предмет делает движение по влажному выступу неопределённым.

Прыжок с полного перепада очевидно опасен.
```

Последствия должны быть однозначными.

Если исход неочевиден, в карточке нужно указать условие проверки и возможные результаты.

Пример:

```
Если персонаж идёт по влажному выступу с большим предметом, исход становится неопределённым: нужна проверка. При провале персонаж теряет равновесие, а предмет цепляется за край выступа или падает вниз.
```

Принцип:

```
Очевидный успех — без кубика.

Очевидный провал — без кубика.

Неопределённость — кубик.
```

## Проход с каждой стороны

Переход описывается с двух сторон.

Формат:

```
## Проход со стороны [[L001_Old_Training_Ledge|Старого учебного уступа]]

...

## Проход со стороны [[L002_Lower_Wet_Ledge|Нижнего влажного уступа]]

...
```

Это нужно потому, что один и тот же переход может ощущаться по-разному в разных направлениях.

Спуск и подъём почти всегда различаются.

В горизонтальных переходах различия могут быть меньше, но всё равно меняется то, что персонажи видят первым.

## Точки прохождения

Каждая направленная часть перехода состоит из точек прохождения.

Пример:

```
## Проход со стороны [[L001_Old_Training_Ledge|Старого учебного уступа]]

### Верхний край стены

...

### Влажный выступ

...

### Нижняя стенка

...
```

Точка прохождения — это не просто объект. Это небольшой фрагмент движения.

В точке может быть:

```
атмосферное описание;
флора;
следы фауны;
риск;
скрытый переход;
находка;
ручей;
верёвка;
старый знак;
следы предыдущих Delvers;
место для остановки;
изменение состояния перехода.
```

Точка может иметь 0 интересностей и работать только как атмосфера.

Точка может иметь несколько интересностей, если это важный участок.

## Формат точки прохождения

```
### Название точки

Описание того, что персонажи видят и ощущают в этой части перехода.

Если персонажи совершают конкретное действие, описывается реакция среды или последствия.

**Для мастера:** краткая расшифровка: что здесь скрыто, какие условия, какие последствия.
```

Пример:

```
### Верхний край стены

Путь начинается у неровного края уступа. Камень здесь сухой только сверху: ниже по стене он темнеет, а старая верёвка уходит вдоль трещины, не привлекая к себе внимания сразу. Рядом с местом крепления лежит мелкая каменная крошка, будто зацеп уже несколько раз сдвигался под нагрузкой.

Если персонажи проверяют край перед спуском, они замечают, что верёвка не столько держит путь, сколько показывает старый маршрут.

**Для мастера:** верёвка ненадёжна. Резкая нагрузка повреждает зацеп. Без верёвки путь всё равно существует, но требует аккуратного движения по выступам.
```

## Асимметричные детали

Некоторые детали могут быть заметны только при движении с одной стороны.

Это особенно полезно для лабиринтов, тоннелей, каверн, вертикальных проходов и обратного пути.

Пример:

```
### Узкий проём

Со стороны входа проём выглядит как обычная тёмная щель между камнями. Потолок низкий, а стены закрывают верхнюю часть прохода.

**Для мастера:** если персонажи идут обратно, они легче замечают гнездо мелкой фауны под потолком у проёма. При движении внутрь его можно заметить только при специальном осмотре потолка.
```

Использовать этот приём осторожно. Он хорош как редкая деталь, но не должен превращать каждый переход в игру “угадай, куда посмотреть”.

## Альтернативные состояния внутри точек

Альтернативы пишутся там, где они применяются.

Используем стандартные callouts Obsidian.

Пример:

```
> [!info] Ночью
> Нижняя площадка почти не видна, но влажная поверхность слабо блестит.

> [!danger] После дождя
> Вода стекает по камню, и выступ перестаёт быть надёжной точкой остановки.
```

Базовое описание остаётся обычным текстом. Callouts нужны только для альтернатив.

## Общие альтернативные состояния

Ближе к концу карточки можно оставить короткую памятку альтернатив, если они ещё не расписаны полностью.

Пример:

```
## Альтернативные состояния

Ночью: хуже видна нижняя площадка.  
После дождя: влажный выступ становится опасным.  
После повреждения: старая верёвка больше не используется как опора.
```

Если альтернативы уже подробно описаны в точках прохождения, здесь можно оставить только краткое напоминание.

## Связи

Блок связей всегда находится в самом низу карточки.

Пример:

```
## Связи

[[L001_Old_Training_Ledge]]  
[[L002_Lower_Wet_Ledge]]
```

Если переход соединяется с другим переходом, его тоже указываем:

```
[[T004_Narrow_Side_Crack]]
```

```
# T001 Old Rope Drop (Старый верёвочный спуск)

Высота: между 120 м и 145 м от поверхности.

Перепад: около 25 м.

Соединяет [[L001_Old_Training_Ledge]] и [[L002_Lower_Wet_Ledge]].

## Кратко

Переход между двумя уступами. Сверху вниз он ощущается как быстрый спуск, снизу вверх — как медленный подъём по влажной стене. Старая верёвка помогает читать маршрут, но не является надёжной страховкой.

## Для мастера

Верёвка ненадёжна. Большой предмет мешает на влажном выступе. Если повредить верхний край или оборвать верёвку, обратный путь станет опаснее.

## Состояние перехода

- [ ] Верёвка оборвана.
- [ ] Верхний край осыпался.
- [ ] Влажный выступ стал скользким.
- [ ] Переход привлёк мелкую фауну.

## Опорная логика перехода

Старая верёвка показывает прежний маршрут, но не выдерживает резкой нагрузки.

Собственная верёвка помогает, если персонажи нашли надёжное крепление.

Большой предмет делает движение по влажному выступу неопределённым.

Прыжок с полного перепада очевидно опасен.

## Проход со стороны [[L001_Old_Training_Ledge|Старого учебного уступа]]

### Верхний край стены

Путь начинается у неровного края уступа. Камень здесь сухой только сверху: ниже по стене он темнеет, а старая верёвка уходит вдоль трещины, не привлекая к себе внимания сразу. Рядом с местом крепления лежит мелкая каменная крошка, будто зацеп уже несколько раз сдвигался под нагрузкой.

Если персонажи проверяют край перед спуском, они замечают, что верёвка не столько держит путь, сколько показывает старый маршрут.

**Для мастера:** верёвка ненадёжна. Резкая нагрузка повреждает зацеп. Без верёвки путь всё равно существует, но требует аккуратного движения по выступам.

### Влажный выступ

На середине спуска камень выдаётся небольшой тёмной полкой. Здесь можно на миг остановиться, но поверхность прохладная и влажная, а под ногами видна узкая трещина с мхом.

> [!danger] После дождя
> Вода стекает прямо по выступу. Он перестаёт быть надёжной точкой остановки.

**Для мастера:** персонаж с большим предметом двигается здесь с риском. При осмотре трещины можно заметить слабый сквозняк или боковое ответвление.

### Нижняя стенка

Последние метры ниже и уже. Стена становится менее крутой, но влажнее, а камень у основания покрыт тёмными пятнами. Нижний уступ виден почти полностью, хотя часть поверхности закрыта выступающим камнем.

**Для мастера:** это безопаснее верхнего участка, если персонажи уже контролируют спуск. При шумном движении можно привлечь мелкую фауну снизу.

## Проход со стороны [[L002_Lower_Wet_Ledge|Нижнего влажного уступа]]

### Нижняя стенка

Снизу переход выглядит выше, чем казался сверху. Камень поднимается неровной влажной стеной, а старая верёвка теряется среди трещин и выступов. Нижняя часть стены даёт несколько зацепов, но они мокрые и неудобные.

**Для мастера:** подъём медленнее спуска. Большой предмет делает первые метры неудобными.

### Влажный выступ

Снизу выступ заметен лучше: он нависает тёмной полкой, под которой держится прохладный воздух. На краю выступа виден мох, а в боковой трещине можно заметить слабое движение воздуха.

**Для мастера:** при движении снизу вверх персонажи легче замечают боковую трещину, чем при спуске сверху.

### Верхний край стены

Верхний край с этой стороны выглядит сухим и ломким. Старая верёвка натянута неровно, а место крепления видно только в последние метры подъёма.

Если персонажи тянут верёвку снизу, сверху может осыпаться мелкая каменная крошка.

**Для мастера:** если зацеп уже повреждён, подъём по старой верёвке невозможен или очевидно опасен.

## Альтернативные состояния

Ночью: хуже видна нижняя площадка.  
После дождя: влажный выступ становится опасным.  
После повреждения: старая верёвка больше не используется как опора.

## Связи

[[L001_Old_Training_Ledge]]  
[[L002_Lower_Wet_Ledge]]
```

## Краткое правило заполнения

```
Переход не должен ограничивать игроков списком действий.

Описание показывает путь.

Точки прохождения ведут сцену.

Опорная логика объясняет последствия.

Состояние перехода фиксирует изменения.

Две стороны перехода помогают показать, что путь туда и обратно ощущается по-разному.
```
